# Theme customization

Theme definitions live in `src/features/theme/theme.config.ts`. Shared types live in `src/features/theme/theme.types.ts`; CSS token output is generated by `src/features/theme/theme.utils.ts` and injected by `ThemeStyles`.

## Add a custom theme

1. Add a new id to `CustomThemeId` in `theme.types.ts`.
2. Add a `CustomThemeDefinition` object to `customThemeDefinitions` in `theme.config.ts`.
3. Set `enabled: true` to expose it in the visitor theme selector.
4. Optionally set `activeTheme.defaultSelection` to make it the default.

Custom themes also declare availability. Base modes and `dotmoe` remain available; seasonal themes can be manual or Gregorian date windows.

```ts
export const sakuraTheme = {
  id: "sakura",
  label: "Sakura Build",
  description: "Soft seasonal workspace for spring experiments.",
  enabled: true,
  availability: { type: "manual", enabled: true },
  tokens: {
    background: "250 244 248",
    foreground: "32 28 33",
    surface: "244 235 241",
    surfaceMuted: "238 226 235",
    border: "211 184 204",
    accent: "185 92 143",
    accentMuted: "242 210 230",
    secondaryAccent: "137 117 205",
    selection: "236 200 224",
    gridOpacity: "0.34",
    decorationOpacity: "0.18",
  },
  metadata: {
    sidebarStatus: "SPRING BUILD",
    sidebarThemeLabel: "SAKURA BUILD",
  },
  decoration: { enabled: true, type: "blossoms", intensity: "low" },
  particles: {
    enabled: true,
    preset: "blossoms",
    disableOnMobile: true,
    respectReducedMotion: true,
    options: {
      particles: {
        number: { value: 12 },
        opacity: { value: 0.14 },
      },
    },
  },
};
```

## Availability windows

Use `availability` in `theme.config.ts`:

```ts
availability: { type: "always" }
availability: { type: "manual", enabled: false }
availability: {
  type: "gregorian-range",
  timezone: "Asia/Ho_Chi_Minh",
  start: { month: 12, day: 1 },
  end: { month: 1, day: 7 },
}
```

Winter Deploy uses the Gregorian range above and supports ranges crossing the year boundary. Tết Build is manual for now:

```ts
availability: { type: "manual", enabled: false }
```

Temporarily set `enabled: true` for Tết testing. A future `vietnamese-lunar-range` adapter can be added in `theme.availability.ts` without changing the selector UI.

Development preview override:

```text
NEXT_PUBLIC_THEME_PREVIEW=winter
NEXT_PUBLIC_THEME_PREVIEW=tet
```

Preview is ignored in production.

To expose every `enabled: true` custom theme even in production, set:

```text
NEXT_PUBLIC_THEME_ENABLE_ALL=true
```

This affects both the selector availability check and the early no-flash theme bootstrap script, so a persisted seasonal theme such as `tet` can be restored immediately on first paint.

## Tokens

Tokens are RGB triplets as strings, for example `"13 16 18"`. Keep contrast high for MDX prose. If a theme needs light/dark-aware behavior, either define a fixed custom theme palette or add theme-scoped CSS in `atmosphereCss` with `prefers-color-scheme` media queries.

## Metadata and decoration

Use `metadata.sidebarStatus` and `metadata.sidebarThemeLabel` for sidebar/drawer microcopy. Use `atmosphereCss` for theme-scoped background glows. Keep it behind content and avoid gradients under long-form text.

## Particles

Set `particlesEnabled` in `theme.config.ts` to disable all particles globally. Each theme can use a simple preset:

```ts
particles: {
  enabled: true,
  preset: "snow",
  disableOnMobile: true,
  respectReducedMotion: true,
}
```

Particle code is lazy-loaded. The normal `system`, `light`, `dark`, and non-particle paths do not import the tsParticles canvas module. `ThemeDecoration` dynamically loads `ParticleCanvas` only when the active theme has `particles.enabled: true` and the global `particlesEnabled` flag is on.

For advanced themes, add a tsParticles `options` object. Safe defaults are merged in `ParticleCanvas`; full-screen background placement and disabled links are forced so particles stay behind content and never capture interaction.

## Testing

Test every theme in desktop sidebar, mobile drawer, light/system/dark color modes, and with `prefers-reduced-motion: reduce`. Check long notes, project case studies, code blocks, tables, language controls, and menu focus states. Mobile particles should be disabled unless the effect is extremely cheap.

## Placeholder media

Reusable placeholders live in `public/content/placeholders`:

```text
project-icon-placeholder.svg
project-cover-placeholder.svg
note-cover-placeholder.svg
```

Replace these files to update static fallback media. Do not generate fallback images from titles, tags, random values, or hashes.

## Accessibility and performance

Use high-contrast tokens, keep motion optional, respect reduced motion, cap particle counts, avoid blur-heavy effects, and keep decorative layers `pointer-events: none`. Media must include meaningful alt text in MDX frontmatter when it is content-bearing.
